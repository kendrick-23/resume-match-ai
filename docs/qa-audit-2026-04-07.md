# Holt Frontend QA Audit — 2026-04-07

Full mobile + desktop audit conducted against the React frontend at this commit.
Findings are read-only — no files were modified during the audit. Diagnosis is
based on reading every screen, layout component, and CSS token file.

---

## CRITICAL — broken or unusable on mobile

**`Dashboard.jsx:128-134` — "Applications logged" wraps awkwardly on iPhone 14 Pro and Android narrow viewports**

Today's Summary grid uses `gridTemplateColumns: '1fr 1fr'` with `gap: var(--space-3)` (12px). Each tile is a `<Card>` (default padding `var(--space-5)` = 20px). Container math at 390px viewport:

- ScreenWrapper L+R padding: 32px → effective grid width 358px
- Per-column: `(358 − 12) / 2` = **173px**
- Card usable width: `173 − 40` = **133px**
- "Applications logged" at Nunito 600 / 13px ≈ 95–110px nominal — sits right at the wrap edge once Safari's font metrics + sub-pixel rounding are factored in.

At **360px** (older Android / Pixel 7 narrow): per-column drops to ~161px, usable ~121px → **wrap is guaranteed**, not borderline.

Root cause is geometric, not a font bug: 40px of card padding inside a 173px column is half the column. The label is intentionally short ("Applications logged" vs. "Applications today" used on the other tile, which is 17 chars vs. 21) and still loses.

## HIGH — noticeably degraded experience

- **No desktop max-width anywhere.** `tokens.css` defines `--max-width-desktop: 1200px` but it's never used. `ScreenWrapper` clamps to `--max-width: 480px`. On a 1280px+ monitor every screen is a narrow 480px column floating against ~400px of dead space on the right. Affects all 9 screens.
- **Auth screens (`Login.jsx:42`, `Signup.jsx:54`) have no desktop layout.** Ott splash hardcoded to `width: 160px`. At 360px Android that's 44% of viewport (cramped); at 1280px desktop it's 12.5% (lost). Same hardcoding pattern in `Onboarding.jsx` (200px) and `Upload.jsx:178` (`ott-reading.png` 160px).
- **Score ring undersized on desktop.** `Results.jsx` `ScoreRing` uses fixed `RING_SIZE = 140`. CLAUDE.md calls for "dramatic" reveal animation; at 140×140 on a 1280px monitor it occupies ~11% width. The reveal moment loses impact.
- **Pill nav on Results forces horizontal scroll on 360px.** 5 pills (Overview, Strengths, Gaps, Ott's Take, Resume) at 13px text + min-height 36px need ~320px just for text+padding. Scroll bar hidden via `scrollbar-width: none` so users have no indication "Resume" is reachable by swiping.
- **Pill `min-height: 36px` is below the 44px accessibility minimum** specified in CLAUDE.md design system. (Buttons at 52px, Inputs at 48px are fine; this one regression breaks the floor.)
- **`Profile.jsx` form has no desktop reflow.** All fields stack as a single 480px column even at 1280px. With 10+ fields the form runs vertically forever in the middle of empty space.

## MEDIUM — polish issues

- **`Tracker.jsx:174-188` "Add" button uses inline-style `padding: 10px 20px, minHeight: 44px, fontSize: 14px`** instead of the standard primary `Button` component. Inconsistent typography (14px vs Button's 16px) and no 3D press shadow.
- **Empty states are vertically heavy on mobile.** `Dashboard.jsx` `<EmptyStateDashboard size={180}>` consumes ~21% of an iPhone 14 Pro viewport before any text. `Results.jsx:410` empty card uses `padding: var(--space-10) var(--space-5)` = 40px×20px → ~80px of internal whitespace before content. Combined with a 120px Ott, the empty card eats ~23% of vertical space.
- **Header tightness on 360px.** `ScreenWrapper.css` header is 56px tall with HoltWordmark (logo 36px + "holt" wordmark) + screen name + 40×40 settings icon. At 360px the wordmark + name pair has ~16px of breathing room before the settings icon — visually crowded.
- **Stagger animation cap.** `animations.css:26-37` targets `.screen__content > *:nth-child(N)` up to N=12. Results.jsx renders 13+ direct children when all sections are open; the 13th+ has no entrance delay and pops in flat.
- **Button vs Input min-height inconsistency.** Button = 52px, Input = 48px. Both exceed the 44px floor but the visual rhythm is off when buttons and inputs sit adjacent in a form.
- **No lazy loading on splash images.** `ott-splash.png` and `ott-reading.png` (each ~600KB+ at 1024×1024) are eager-loaded on Auth/Onboarding/Upload. No `loading="lazy"` attribute anywhere in the JSX.

## LOW — nice to have

- **No Android back button handling anywhere** (no `popstate` listener, no React Router programmatic back). Android users get default browser back, which works but doesn't integrate with the in-app modals (e.g., Profile delete confirmation, Tracker add/edit sheets).
- **Bottom nav z-index** appears to be `100` (BottomNav). Adequate vs. Card content but could collide with future modal overlays — Toast and MilestoneCelebration overlays should be checked for z-index above 100.
- **Safe-area-inset-bottom is honored** in `ScreenWrapper.css:9` (`calc(var(--bottom-nav-height) + var(--space-4) + env(safe-area-inset-bottom, 0px))`). Verified present. Good.
- **No code-splitting / `React.lazy`.** All 9 screens ship in the main bundle. At Holt's current size (~50 components) this is acceptable but worth flagging for first-load performance on 3G.
- **Profile delete account inline confirmation** scrolls into the same column with no modal overlay. Easy to miss on desktop; on mobile it pushes the "real" delete button below the fold without scroll-to.
- **Badge labels use `white-space: nowrap`** in `Badge.jsx` — no overflow guard for unusually long labels (not currently a problem since all 8 badge names are ≤14 chars, but a latent vuln).
- **No visual indicator of the 480px mobile column on desktop.** No left/right border, no background tint shift. Desktop users may be confused why the page is so narrow.
- **`Jobs.jsx` job cards render single-column at all viewports.** Even at 1200px there's no 2- or 3-column grid for results; just a tall list in a 480px column.
- **EmptyState illustrations not lazy-loaded.** `EmptyStateDashboard`, `EmptyStateJobs`, `EmptyStateUpload` (where still imported) load eagerly.

## AUDIT 5 — Performance quick check (specific findings)

- **Dashboard mount fires 3 sequential network calls:** `useStreak()` hook (1 call), `getActivity()` (1 call), `listBadges()` (1 call). They are not bundled into a `Promise.all` — worth flagging as a HIGH if the user cares about TTI, but classed as MEDIUM here because all 3 are fast.
- **All 23 PNGs in `frontend/public/ott/`** are referenced via plain `<img src="/ott/...">` tags scattered across components. Browsers will only fetch them when each component mounts (the static `src` is not pre-fetched), so there is **no eager loading of the full set** — but there's also no `<link rel="preload">` on the critical first-screen Ott (`ott-splash.png` on Login). First auth load fetches it inline.
- **No `React.lazy` / `React.Suspense`** anywhere. All 9 screens + 30+ components are in the initial chunk.
- **No memoization** on Dashboard's `streakLabel` / `streakHint` / Profile's heavy filter loops — recomputed every render. Minor.
- **No service worker / PWA caching wired up** despite CLAUDE.md flagging PWA as a hard requirement (`manifest.json` exists; no SW registration in `main.jsx`). Flagged in CLAUDE.md as a "still to build" item.

## Next Session Fix Plan

### PRIORITY 1 — CRITICAL (fix first)

- **"Applications logged" label → change to "Apps logged"** in `Dashboard.jsx:128-134`. Shorter label fits the 121px usable card width on 360px Android even with sub-pixel rounding. Lowest-risk single-line change.

### PRIORITY 2 — HIGH (one CSS session)

- **Add responsive breakpoints to `ScreenWrapper.css`:** `@media (min-width: 768px)` (tablet) and `@media (min-width: 1200px)` (desktop). Raise `max-width` at each tier and switch select grids to multi-column. Resolves 4 of 6 HIGH findings in one file.
- **Fix Results pill nav `min-height` to 44px** (currently 36px). Restores the accessibility floor specified in CLAUDE.md.
- **Add `loading="lazy"`** to `ott-splash.png` (Login, Signup, Onboarding) and `ott-reading.png` (Upload). Each is ~600KB at 1024×1024.
- **Fix Ott image sizes to be responsive, not hardcoded.** Replace inline `width: 160px` / `width: 200px` with CSS classes that scale via clamp() or viewport-relative units. Affects `Login.jsx`, `Signup.jsx`, `Onboarding.jsx`, `Upload.jsx`.

### PRIORITY 3 — MEDIUM (next pass)

- **Dashboard 3 serial calls → `Promise.all`**. Bundle `useStreak`, `getActivity`, `listBadges` to fire concurrently. Saves ~2× network round-trip on mount.
- **`Tracker.jsx` Add button → use standard `Button` component.** Removes the inline-style fork and restores the 3D press shadow + Nunito 800 / 16px typography consistency.
- **Stagger animation cap → extend to 20 children** in `animations.css:26-37`. Prevents Results sections beyond the 12th from popping in flat.
